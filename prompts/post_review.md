### Инструкция по обработке тест-кейса

# Входные данные

**testcase_id:** `{{testcase_id}}`

**review_result:**
```
{{review_result}}
```

# Форматы ответа ревью

`review_result` может иметь один из трёх форматов:

| Формат | Содержание | Признаки |
|--------|-----------|----------|
| **A** | Полный ревью с замечаниями и правками | Нумерованные секции (1., 2., ...), блок «Предлагаемые правки» |
| **B** | Нет новых замечаний (есть предыдущие) | Фраза «Новых замечаний ... нет» |
| **C** | Замечаний нет вообще | Фраза «Замечаний по ревью нет» |

# Алгоритм действий:

1. **Проверка тест-кейса:**
   - Запросить тест-кейс из Allure TestOps по ID
   - Если не существует → вернуть ошибку "Тест-кейс не найден"
2. **Определение формата ответа:**
   - Если `review_result` содержит нумерованные секции → **Формат A** (есть замечания)
   - Если `review_result` содержит фразу «Новых замечаний» → **Формат B** (нет новых замечаний)
   - Если `review_result` содержит фразу «Замечаний по ревью нет» → **Формат C** (замечаний нет)
3. **Публикация комментария:**
   - Опубликовать `review_result` дословно (для любого формата A, B или C)
4. **Изменение статуса:**
   - Если **Формат A** или **B** (есть замечания) → установить статус «Требует актуализации»
   - Если **Формат C** (замечаний нет вообще) → установить статус «Готов к автоматизации»
   - Если статус уже установлен и совпадает → оставить без изменения
