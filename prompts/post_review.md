### Инструкция по обработке тест-кейса

# Входные данные

**testcase_id:** `{{testcase_id}}`

**review_result:** 
```
{{review_result}}
```

# Алгоритм действий:

1. **Проверка тест-кейса:**
   - Запросить тест-кейс из Allure TestOps по ID
   - Если не существует → вернуть ошибку "Тест-кейс не найден"
2. **Анализ содержания ревью:**
   - Если `review_result` содержит текст (не пустой) → **публиковать комментарий**
3. **Публикация комментария:**
   - Опубликовать `review_result` дословно
   - Сохранить ID созданного комментария
4. **Изменение статуса:**
   - Проанализируй `review_result` на наличие замечаний:
     - Признаки замечаний: нумерованные списки, блоки "Что не так", "Пример исправления", названия разделов (Заголовок, Описание, Сценарий и т.д.)
     - Отсутствие замечаний: текст пустой, содержит только "Всё ок", "Замечаний нет", "Тест-кейс соответствует требованиям"
   - Если есть замечания → установить статус «Требует актуализации»
   - Если статус уже установлен → оставить без изменения
   - Если нет замечаний → статус не менять
5. **Возврат результата:**
   - Вернуть ответ в формате ключ-значение (каждое поле на новой строке, без markdown, без JSON).

   **Успех (статус изменён):**
   ```
   success: true
   testcase_id: 80719
   comment_id: 12345
   status_updated: true
   message: Ревью опубликовано, статус изменён
   ```

   **Успех (статус уже был установлен):**
   ```
   success: true
   testcase_id: 80719
   comment_id: 12345
   status_updated: false
   message: Ревью опубликовано, статус уже установлен
   ```

   **Успех (нет замечаний, статус не менялся):**
   ```
   success: true
   testcase_id: 80719
   comment_id: 12345
   status_updated: false
   has_remarks: false
   message: Ревью опубликовано, замечаний нет
   ```

   **Ошибка:**
   ```
   success: false
   error: Тест-кейс не найден
   testcase_id: 99999
   ```
