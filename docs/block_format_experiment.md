# Синтетический тест: Как LLM понимает блоки

## Тестовые данные

Содержимое блока:
```
Это текст с **жирным**, *курсивом* и `кодом`.
В нём есть несколько строк.
И даже цифры: 123, 456.
```

---

## Вариант 1. Жирный + перенос

```
**review_result:**
Это текст с **жирным**, *курсивом* и `кодом`.
В нём есть несколько строк.
И даже цифры: 123, 456.
```

**Вопрос для LLM:** Где заканчивается `review_result`?
**Проблема:** Нет явного конца. Модель может решить, что блок продолжается до следующего `**` или конца текста.

---

## Вариант 2. XML-теги

```
<review_result>
Это текст с **жирным**, *курсивом* и `кодом`.
В нём есть несколько строк.
И даже цифры: 123, 456.
</review_result>
```

**Вопрос для LLM:** Где заканчивается `review_result`?
**Ответ:** Ясно — после `</review_result>`

---

## Вариант 3. Код-блок

```
```
review_result:
Это текст с **жирным**, *курсивом* и `кодом`.
В нём есть несколько строк.
И даже цифры: 123, 456.
```
```

**Вопрос для LLM:** Где заканчивается?
**Ответ:** После закрывающих ```

---

## Вариант 4. Разделители

```
>>> review_result
Это текст с **жирным**, *курсивом* и `кодом`.
В нём есть несколько строк.
И даже цифры: 123, 456.
<<<
```

**Вопрос для LLM:** Где заканчивается?
**Ответ:** После `<<<`

---

## Вариант 5. Квадратные скобки

```
[review_result]
Это текст с **жирным**, *курсивом* и `кодом`.
В нём есть несколько строк.
И даже цифры: 123, 456.
[/review_result]
```

**Вопрос для LLM:** Где заканчивается?
**Ответ:** После `[/review_result]`

---

## Анализ пограничных случаев

### Случай A: В тексте есть `**`

Содержимое: `Текст с **жирным** выделением`

| Формат | Путаница? |
|--------|-----------|
| `**name:**` | ДА — модель может подумать, что это конец блока |
| `<name>...</name>` | НЕТ — теги уникальны |
| `>>>...<<<` | НЕТ |
| `[name]...[/name]` | НЕТ |

### Случай B: В тексте есть похожие теги

Содержимое: `Проверка <test> значения`

| Формат | Путаница? |
|--------|-----------|
| `<name>...</name>` | ВОЗМОЖНО — если содержит `<name>` |
| `[name]...[/name]` | НЕТ — квадратные скобки реже |

### Случай C: Многострочный текст

Содержимое: 100 строк текста

| Формат | Надёжность |
|--------|------------|
| `**name:**` | НИЗКАЯ — нет явного конца |
| `<name>...</name>` | ВЫСОКАЯ |
| `` ``` `` `` | ВЫСОКАЯ |
| `>>>...<<<` | ВЫСОКАЯ |
