# 2. Описание тест-кейса

## Формат входных данных

```
2. Описание
   - Заменить описание на: «Новый текст описания»

Или:

2. Описание
   - Добавить в описание: «Дополнительный текст»
   - Удалить из описания: «Устаревший текст»
```

## Алгоритм

1. Извлечь новый текст описания из кавычек
2. Проверь наличие участников/ссылок для добавления
3. Если есть участники → вызови nested skills для members
4. Если есть ссылки → вынеси их в поля links/issues
5. Удали вынесенные ссылки из текста
6. Вызови `testcase_patch` с обновлённым `description`

## Особенности

- **Вложенные скиллы**: edit_description автоматически вызывает edit_members и edit_links
- **Участники**: форматы `Роль: ФИО`, `Роль - ФИО @telegram`, `[ФИО](ссылка)`
- **Ссылки**: markdown-ссылки вида `[Название](URL)` выносятся в `links`

## MCP инструменты

```python
mcp__allure-http__testops_testcase_patch(
    testcase_id=80532,
    description="Новый текст описания"
)
```
