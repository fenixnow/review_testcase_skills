# Тесты для скилла testops-edit-testcase

## Назначение

Тесты проверяют корректность работы скилла редактирования тест-кейсов, особенно при работе с **вложениями в шагах**.

## Критический тест: Удаление шага с вложением

**Проблема:** Ранее скилл удалял шаги без проверки вложений, что приводило к потере важных данных (примеры ответов API, cURL-запросы, JSON-структуры).

**Решение:** Теперь скилл:
1. Проверяет вложения перед удалением шага
2. Предупреждает пользователя о наличии вложений
3. Не удаляет шаг без явного подтверждения

## Запуск тестов

### Вручную через Claude Code:

```bash
# Тест 1: Удаление шага с вложением
cd /Users/fenixnow/IdeaProjects/review_testcase_skills
python3 -c "
import json
with open('skills/testops-edit-testcase/evals/evals.json') as f:
    evals = json.load(f)
    for eval in evals['evals']:
        print(f\"Тест {eval['id']}: {eval['name']}\")
        print(f\"  TestCase: {eval['testcase_id']}\")
        print(f\"  Expected: {eval['expected_output']}\")
        print()
"
```

## Тест-кейсы

| ID | Название | Проверяет | Ожидаемый результат |
|----|----------|-----------|---------------------|
| 1 | Удаление шага с вложением | Защиту вложений при удалении | Шаг НЕ удалён, пользователю предложены альтернативы |
| 2 | Удаление пустого шага | Удаление без вложений | Шаг успешно удалён |
| 3 | Перемещение шага с вложением | Сохранение вложений | Шаг перемещён, вложение сохранено |

## Подготовка тестовых данных

Для запуска тестов нужны тестовые тест-кейсы в Allure TestOps:

- **#99999** — Шаг 3 с вложением «Пример ответа API»
- **#99998** — Пустой шаг 5 без вложений
- **#99997** — Шаг 2 с вложением «cURL запроса»

## Assertions

Тесты проверяют следующие утверждения:

### Тест 1:
- `skill_checked_attachments_before_delete` — скилл проверил вложения
- `step_not_deleted` — шаг не удалён
- `warning_about_attachments` — выдано предупреждение

### Тест 2:
- `skill_checked_attachments` — проверка выполнена
- `step_deleted` — шаг удалён

### Тест 3:
- `step_moved` — шаг перемещён
- `attachment_preserved` — вложение сохранено
