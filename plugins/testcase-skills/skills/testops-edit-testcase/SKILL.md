# Edit Testcase in Allure TestOps

Специалист по редактированию тест-кейсов в Allure TestOps на основе результатов ревью.

## Параметры

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `testcase_id` | int | Да | ID тест-кейса |
| `proposed_edit` | str | Да | Полный текст "Предлагаемых правок" из ревью |

## Алгоритм

```
1. Получить текущее состояние тест-кейса (testcase_get)
   ↓
2. Определить какие секции присутствуют в proposed_edit
   ↓
3. Для каждой секции — выполнить соответствующие изменения
   ↓
4. Подтвердить изменения (проверить результат)
   ↓
5. Сформировать отчёт
```

## Роутинг секций

По наличию заголовков в `proposed_edit` определяю, что менять:

| Секция | Маркер | Действие | См. |
|---------|--------|----------|-----|
| 1. Имя | `1. Имя тест-кейса` | Изменить `name` | [name.md](references/name.md) |
| 2. Описание | `2. Описание` | Изменить `description` | [description.md](references/description.md) |
| 3. Предусловие | `3. Предусловие` | Изменить `precondition` | [precondition.md](references/precondition.md) |
| 4. Ссылки | `4. Ссылки` | Переместить в `links/issues` | [links.md](references/links.md) |
| 5. Участники | `5. Участники` | Добавить в `members` | [members.md](references/members.md) |
| 6. Шаги | `6. Шаги сценария` | Изменить `steps` | [steps.md](references/steps.md) |
| 7. Вложения | `7. Вложения` | Создать `attachments` | [attachments.md](references/attachments.md) |

## Порядок применения

Секции применяю в таком порядке:
1. **links** → добавление в issues/links (без изменения текста)
2. **members** → добавление участников
3. **name** → изменение имени
4. **description** → изменение описания (может удалить ссылки)
5. **precondition** → изменение предусловия
6. **steps** → изменение шагов
7. **attachments** → создание вложений

> **Важно:** Ссылки (`links`) обрабатываю раньше описания (`description`), чтобы description мог корректно удалить уже вынесенные ссылки.

## Извлечение данных из секций

Для каждой секции извлекаю данные по формату:

**Формат 1 — прямое значение:**
```
- Изменить имя на: «Новое название»
```

**Формат 2 — с альтернативой:**
```
- Изменить имя на: «Вариант 1»
- или: «Вариант 2»
```
→ использовать первый вариант

**Формат 3 — участники:**
```
- Добавить участника: username найти по ФИО «Иванов», role_id найти роль «Аналитик»
- Найти и добавить участника: username для Петров, role_id -1 (Owner)
```

## Выходной формат

```
## Результат выполнения:

**Применено изменений:** N

**1. Имя тест-кейса:**
- Было: `старое`
- Стало: `новое`

**2. Участники добавлены:**
- Иванов — Аналитик ✅

**3. Ссылки перемещены:**
- https://... → Связанные задачи ✅

**Изменения подтверждены** в TestOps.
```

## Обработка ошибок

- При ошибке в одной секции — продолжаю обработку остальных
- Критические ошибки (403, 404) — прерываю выполнение
- Результат содержит список успешных и неудачных изменений
